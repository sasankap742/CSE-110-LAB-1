Three So why are CSS stylesheets “cascading”? This is to ensure a definitive, consistent style if there are multiple style references to the same HTML element. You can think of it like the order of operations in mathematics (i.e. calculate exponents before multiplying, multiply before adding, etc.). For example, HTML elements have some default styles, but the CSS that you as a developer specify takes precedence, and any inline CSS that’s more specific to one element takes precedence over your attached stylesheet for multiple elements, and any CSS you designate with an !important takes precedence over pretty much everything. In the “Styles” tab of the inspector tool, you can see the style references for an element – note how the color attributes in the example below are crossed out, since the specified colors at the top took precedence over those styles.
